Was ist WINDI-Governance?

WINDI ist eine Pre-AI Governance Layer — eine Schicht, die vor der KI-Verarbeitung greift. Bevor ein Dokument erstellt wird, prüft WINDI ob alle Governance-Anforderungen erfüllt sind. Fehlen Pflichtfelder, wird das Dokument blockiert.

Das System arbeitet nach dem Prinzip der proportionalen Governance: Je sensibler das Dokument, desto strenger die Prüfung. Ein internes Memo braucht keine forensische Versiegelung. Ein Regulierungsbericht braucht jede Sicherheitsebene.

What is WINDI Governance?

WINDI is a Pre-AI Governance Layer — a system that intervenes before AI processing. Before any document is generated, WINDI validates that all governance requirements are met. If mandatory fields are missing, the document is blocked.

The system works on the principle of proportional governance: the more sensitive the document, the stricter the validation. An internal memo doesn't need forensic sealing. A regulatory report needs every security layer.

O que é Governança WINDI?

WINDI é uma Camada de Governança Pré-IA — um sistema que intervém antes do processamento da IA. Antes de qualquer documento ser gerado, WINDI valida se todos os requisitos estão cumpridos. Se campos obrigatórios estiverem faltando, o documento é bloqueado.

O sistema opera pelo princípio da governança proporcional: quanto mais sensível o documento, mais rígida a validação. Um memorando interno não precisa de selagem forense. Um relatório regulatório precisa de cada camada de segurança.

04 NEU

Tägliches Governance-Briefing

Das Briefing-Tab zeigt den täglichen Governance-Bericht — automatisch generiert aus allen aktiven Diensten. Es enthält eine zweisprachige Zusammenfassung (DE/EN), Metadaten wie Report-ID, Hash, Ledger-Referenz und Systemstatus.

Der Bericht wird vom Day-by-Day Server (:8090) aggregiert und aktualisiert sich automatisch. Er fasst zusammen: wie viele Dokumente in der Audit-Kette sind, welche ISP-Profile geladen sind, ob HIGH-Level Einträge zur Prüfung markiert sind.

Daily Governance Briefing

The Briefing tab shows the daily governance report — automatically generated from all active services. It contains a bilingual summary (DE/EN), metadata such as Report ID, hash, ledger reference and system status.

The report is aggregated by the Day-by-Day Server (:8090) and updates automatically. It summarizes: how many documents are in the audit chain, which ISP profiles are loaded, whether HIGH-level items are flagged for review.

Briefing Diário de Governança

A aba Briefing mostra o relatório diário de governança — gerado automaticamente a partir de todos os serviços ativos. Contém um resumo bilíngue (DE/EN), metadados como ID do relatório, hash, referência do ledger e status do sistema.

O relatório é agregado pelo Day-by-Day Server (:8090) e atualiza automaticamente. Resume: quantos documentos estão na cadeia de auditoria, quais perfis ISP estão carregados, se itens HIGH estão sinalizados para revisão.

Audit & Registry

Die Audit-Seite zeigt das vollständige Register aller Governance-Submissions. Jeder Eintrag enthält: Registration-ID, Datum, Governance-Level, Dokumenttyp, Status und SHA-256 Hash.

Bei HIGH-Dokumenten wird zusätzlich die kryptografische Versiegelung und der Eintrag im Forensic Ledger angezeigt.

Audit & Registry

The Audit page shows the complete registry of all governance submissions. Each entry contains: Registration ID, date, governance level, document type, status and SHA-256 hash.

For HIGH documents, the cryptographic seal and forensic ledger entry are also displayed.

Auditoria & Registro

A página de Auditoria mostra o registro completo de todas as submissões de governança. Cada entrada contém: ID de Registro, data, nível de governança, tipo de documento, status e hash SHA-256.

Identity License System

Identity License System

The Identity License System controls how institutional identities may be used in generated documents. Each ISP profile contains an identity_license with one of five statuses:

Sistema Identity License

O Sistema Identity License controla como identidades institucionais podem ser usadas em documentos gerados. Cada perfil ISP contém um identity_license com um dos cinco status:
