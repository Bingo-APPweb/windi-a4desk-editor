01 Overview

The ISP Manager Agent is the first micro-agent in the WINDI ecosystem. It scans, validates, and proposes corrections for ISP (Institutional Style Profiles) — but NEVER decides or applies changes autonomously (Invariante I9).

04 Data Flow (Zero-Knowledge)

ISP Manager Agent │ ├── READS: /opt/windi/isp/*/profile.json (profile data) ├── READS: /opt/windi/data/template_registry.db (templates) ├── READS: /opt/windi/engine/governance_levels.json (config) │ ├── GENERATES: receipts/ → hash + grade + counts + timestamp ├── GENERATES: alerts/ → code + severity + message (NO sensitive data) ├── GENERATES: drafts/ → correction proposals (JSON patches) ├── GENERATES: reports/ → aggregated health reports │ └── SENDS TO L3: ONLY hash + grade + metadata ZERO sensitive data Client keeps data, WINDI keeps PROOF
🔒 Zero-Knowledge: The agent never transmits sensitive content. Only hashes, grades, and structural metadata leave the client perimeter. Data stays with the client; WINDI stores only the proof of virtue.

06 Client Model

The agent on Strato serves as Template Master. For each new client, the agent ecosystem is cloned with a unique client_id and restricted policy.json.

WINDI Core (Strato) Client "Deutsche Bahn" ┌─────────────────────┐ ┌─────────────────────┐ │ /opt/windi/agents/ │ CLONE │ /client/db/agents/ │ │ └── isp-manager/ │ ──────────► │ └── isp-manager/ │ │ ├── manifest.json│ │ ├── manifest.json│ + client_id │ ├── capsule.json │ │ ├── capsule.json │ │ ├── policy.json │ │ ├── policy.json │ restricted │ └── agent.py │ │ └── agent.py │ └─────────────────────┘ └─────────────────────┘ │ ▼ Virtue Receipt → Forensic Ledger (L3)
💰 Business model: Client does not pay for the agent — pays for the protected decision. The agent generates the proof of virtue (Virtue Receipt). WINDI stores only hashes and metadata — zero sensitive data. Pricing: per protected decision, not storage.
